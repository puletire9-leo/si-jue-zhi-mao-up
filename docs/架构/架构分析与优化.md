## 三、数据管道

```
领星 Excel (550MB ×2)
  → merge_parquet_batch.py
    → product_data_final.parquet (230MB, 150列, 2.57M行)
      → DuckDB SQL 查询（_dedup 去重）
        → 销量追踪 / 双周期对比 / 下滑分析
```

**关键设计：** 所有查询走 `_dedup()` 子查询——按 `(ASIN, 日期, 店铺)` 分组，`MAX()` 取数值列，消除两 Excel 重叠导致的 4 倍重复。

## 五、生产模式启动流程

```
python scripts/startup/start_prod_multiprocess.py
  │
  ├─ 1. 读 production/.runtime.pid → kill 旧进程（仅自己的）
  ├─ 2. 加载 backend/.env.production（覆盖 dev 配置）
  ├─ 3. npm run build（前端构建，如已构建则跳过）
  ├─ 4. 动态生成 Nginx 配置 → 启动 Nginx :8080
  ├─ 5. 启动 uvicorn :8005 (4 workers)
  ├─ 6. 写 production/.runtime.pid
  └─ Ctrl+C → 清进程 + 删 PID 文件
```

### 环境变量隔离

| 配置项 | 开发 (.env) | 生产 (.env.production) |
|--------|------------|----------------------|
| ENVIRONMENT | development | production |
| MYSQL_DATABASE | sijuelishi_dev | sijuelishi |
| COS_BUCKET | sijuelishi-dev-1328246743 | sijuelishi-1328246743 |
| DEBUG | true | false |

`prod.sh` 通过 `set -a; source .env.production` 在启动 uvicorn 前注入生产变量。`load_dotenv('.env')` 默认不覆盖已有环境变量，所以生产配置优先生效。
