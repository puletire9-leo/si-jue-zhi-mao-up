  你缺失的（2026 AI关键能力）

  1. 没有 MCP 协议 — AI 工具调用层的行业标准

  现状：你的 AI 服务（CLIP、百度、腾讯混元）都是硬编码的 REST 调用。

  2026 标准：通过 MCP（Model Context Protocol）暴露工具，AI 模型在运行时动态发现可用的工具，不需要人为预编程调用路径。

  你的现状:  前端 → REST API → service.tencent_llm_service.analyze()
  2026标准:  Agent → MCP Server → Tool Registry → execute(image_analysis_tool)
                      ↑ 运行时发现工具列表

  对你的意义：如果后续要加 AI Agent（自动选品、自动生成定稿描述、自动评分），MCP 是标配。

  ---
  2. 没有 Agent 编排层

  现状：所有 AI 调用都是"单次请求-响应"模式。没有 Agent Loop（规划→执行→观察→调整）。

  2026 标准：采用 Brain/Hands/Session 三层解耦架构（Anthropic 2026年4月发布）：

  Brain (LLM推理) → Hands (工具执行沙箱) → Session (持久状态)

  你的评分引擎 scoring_engine.py 是硬编码规则的，不是 Agent 自主决策的。

  ---
  3. 没有 RAG 检索管道（Guardrails）

  现状：
  # qdrant_repo.py — 纯向量相似搜索，无门控
  results = await qdrant.search(collection_name, query_vector, limit=top_k)

  2026 标准：检索 → 门控 → 重排 → 生成 → 结构校验 的完整管道：

  检索 (Qdrant) → GATE (相似度<阈值→拒绝回答) → Re-rank (交叉编码器)
  → LLM生成 → Pydantic结构校验 → 返回

  你缺少的关键环节：检索门控（搜索质量低时不会告知用户"找不到"）、重排序、结构化输出校验。

  ---
  4. 没有 SSE/WebSocket 流式响应

  现状：全是 HTTP request-response。下载任务用轮询（download_task_service 定期查状态）。

  2026 标准：AI 生成类接口必须用 SSE（Server-Sent Events）流式输出。用户不想等 30 秒才看到 AI 分析结果。

  ---
  5. 没有 AI 可观测性

  现状：
  logger.info(f"[OK] 产品销量API路由注册成功")  # 中文日志，无结构化

  2026 标准：OpenTelemetry + Langfuse/Arize 追踪每一次 LLM 调用：

  ┌────────────┬───────────────────────────────────────┐
  │    指标    │              你应该记录               │
  ├────────────┼───────────────────────────────────────┤
  │ Token 用量 │ 每次 LLM 调用的 input/output token 数 │
  ├────────────┼───────────────────────────────────────┤
  │ 延迟       │ p50/p95/p99 LLM 响应时间              │
  ├────────────┼───────────────────────────────────────┤
  │ 成本       │ 按租户/按功能的 AI 费用分摊           │
  ├────────────┼───────────────────────────────────────┤
  │ 质量       │ 检索结果相似度分布、用户反馈          │
  └────────────┴───────────────────────────────────────┘

  ---
  6. 没有 AI 模型回退/路由（Model Fallback）

  现状：每个 AI 调用直连特定供应商：
  # tencent_llm_vision_service.py — 只用腾讯混元
  # baidu_image_recognition_service.py — 只用百度

  2026 标准：智能模型路由 — 便宜模型做规划，贵模型做最终生成；主模型挂了自动切换备选。

  ---
  7. Prompt 管理散落各处

  现状：AI prompt 大概率是代码里的字符串拼接，没有版本管理，没有 A/B 测试。

  2026 标准：Prompt 作为独立资源管理（版本化、可回滚、可评估）。
