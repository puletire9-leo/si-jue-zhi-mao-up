## TODO-1: 微服务实现

> 目标：从单体 Spring Boot 平滑迁移到微服务架构，5 个独立服务 + Gateway + Nacos

### 架构目标

```
Spring Cloud Gateway :80
  ├─ 用户服务 :8001  (auth/user/RBAC)
  ├─ 产品服务 :8002  (product/selection/scoring/data-pipeline)
  ├─ 定制服务 :8003  (final-draft/material/carrier/category/tag)
  ├─ 图片服务 :8004  (image/COS/AI-vector)
  └─ 公共模块         (Result/JWT/Redis/注解)
      │
      ├─ MySQL (各服务独立 schema)
      ├─ Redis (共享)
      ├─ RocketMQ (异步通信)
      └─ Qdrant (图片向量)
```

- 每切一个服务独立验证，上一个跑稳了再切下一个
- 测试环境先跑，验证通过再切生产
- 数据库不急于拆分——先拆应用层，schema 后续再独立
- 保留 Python 后端不变——只拆 Java 层，FastAPI 图片服务不动
