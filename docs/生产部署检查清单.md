## 二、安全配置

### 2.1 密钥配置 ⚠️ 必做

```bash
# 生成强随机密钥
openssl rand -base64 32
openssl rand -base64 64

# 密钥清单
JWT_SECRET=           # JWT签名密钥 (至少64位)
SECRET_KEY=           # 通用加密密钥 (至少32位)
MYSQL_PASSWORD=       # MySQL应用密码 (至少16位)
REDIS_PASSWORD=       # Redis密码 (至少16位)
COS_SECRET_ID=        # 腾讯云子账号AccessKey
COS_SECRET_KEY=       # 腾讯云子账号SecretKey
```

### 2.2 数据库安全

```sql
-- 1. 创建应用专用账号 (禁止使用root)
CREATE USER 'sjzm_app'@'%' IDENTIFIED BY '你的强密码';
GRANT SELECT, INSERT, UPDATE, DELETE ON sijuelishi.* TO 'sjzm_app'@'%';

-- 2. 限制连接数
SET GLOBAL max_connections = 500;

-- 3. 启用慢查询日志
SET GLOBAL slow_query_log = 1;
SET GLOBAL long_query_time = 2;
SET GLOBAL slow_query_log_file = '/var/log/mysql/slow.log';

-- 4. 定期备份
CREATE EVENT mysql_backup_event
ON SCHEDULE EVERY 1 DAY
DO CALL mysql_backup_procedure();
```

### 2.3 Redis 安全

```conf
# /etc/redis/redis.conf
requirepass 你的强密码
bind 127.0.0.1
protected-mode yes
port 6379
maxmemory 4gb
maxmemory-policy allkeys-lru
appendonly yes
appendfsync everysec
```

### 2.4 防火墙配置

```bash
# Ubuntu (ufw)
ufw default deny incoming
ufw allow 22/tcp    # SSH
ufw allow 80/tcp    # HTTP
ufw allow 443/tcp   # HTTPS
ufw allow 8080/tcp  # Java后端 (仅内网)
ufw enable

# CentOS (firewalld)
firewall-cmd --permanent --add-port=22/tcp
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --reload
```

### 2.5 CORS 配置

```yaml
# application.yml

  cors:
    allowed-origins:
      - https://yourdomain.com
      - https://www.yourdomain.com
    allowed-methods: GET,POST,PUT,DELETE,OPTIONS
```

## 五、监控告警配置

### 5.1 Prometheus 告警规则

```yaml
# prometheus/alerts.yml
groups:
  - name: sjzm-alerts
    rules:
      # CPU 告警
      - alert: HighCPUUsage
        expr: container_cpu_usage_seconds_total > 0.8
        for: 5m
        labels:
          severity: warning
        annotations:
          summary: "服务器 CPU 使用率过高"

      # 内存告警
      - alert: HighMemoryUsage
        expr: container_memory_usage_bytes / container_spec_memory_limit_bytes > 0.85
        for: 5m
        labels:
          severity: warning

      # JVM 告警
      - alert: JVMHeapUsageHigh
        expr: jvm_memory_used_bytes{area="heap"} / jvm_memory_max_bytes{area="heap"} > 0.9
        for: 5m
        labels:
          severity: critical

      # API 响应时间
      - alert: HighLatency
        expr: histogram_quantile(0.99, rate(http_server_requests_seconds_bucket[5m])) > 0.5
        for: 3m
        labels:
          severity: warning

      # 错误率
      - alert: HighErrorRate
        expr: rate(http_server_requests_seconds_count{status=~"5.."}[5m]) > 0.01
        for: 2m
        labels:
          severity: critical

      # 服务宕机
      - alert: ServiceDown
        expr: up == 0
        for: 1m
        labels:
          severity: critical
```

### 5.2 必监控指标

| 指标 | 阈值 | 告警级别 | 处理 |
|------|------|----------|------|
| CPU 使用率 | >80% | 警告 | 扩容 |
| 内存使用率 | >85% | 警告 | 扩容 |
| JVM Old GC | >10次/分 | 严重 | 优化GC |
| API P99延迟 | >500ms | 警告 | 排查慢查询 |
| 错误率 | >1% | 严重 | 立即排查 |
| QPS | 突降50% | 严重 | 立即排查 |
| DB连接数 | >80%上限 | 警告 | 限流 |
| Redis内存 | >90% | 警告 | 清理缓存 |

## 六、数据备份

### 6.1 数据库备份脚本

```bash
#!/bin/bash
# backup.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/data/backup/mysql"
KEEP_DAYS=30

# 创建备份目录
mkdir -p ${BACKUP_DIR}

# 备份数据库
mysqldump -h localhost -u root -p${MYSQL_ROOT_PASSWORD} \
  --single-transaction \
  --routines \
  --triggers \
  --events \
  sijuelishi > ${BACKUP_DIR}/sijuelishi_${DATE}.sql

# 压缩
gzip ${BACKUP_DIR}/sijuelishi_${DATE}.sql

# 删除30天前的备份
find ${BACKUP_DIR} -name "*.sql.gz" -mtime +${KEEP_DAYS} -delete

# 上传到对象存储 (可选)
# aws s3 cp ${BACKUP_DIR}/sijuelishi_${DATE}.sql.gz s3://your-bucket/backup/

echo "[$(date)] 备份完成: sijuelishi_${DATE}.sql.gz"
```

### 6.2 备份频率

| 类型 | 频率 | 保留时间 | 存储位置 |
|------|------|----------|----------|
| 全量备份 | 每天凌晨2点 | 30天 | 本地 + 对象存储 |
| 增量备份 | 每小时 | 7天 | 对象存储 |
| 事务日志 | 实时 | 7天 | 对象存储 |

## 七、部署流程

### 7.1 部署前检查

```bash
# 1. 检查服务器状态
df -h                          # 磁盘空间
free -h                        # 内存
top -bn1 | head -5            # CPU
docker ps                      # 容器状态

# 2. 检查端口占用
netstat -tlnp | grep -E '80|443|3306|6379|8080'

# 3. 检查依赖服务
mysql -h localhost -u root -p -e "SELECT 1"
redis-cli -a 你的密码 ping
```

### 7.2 部署步骤

```bash
# 1. 拉取最新代码
git pull origin main

# 2. 构建前端
cd frontend && npm run build

# 3. 构建 Java 后端
cd ../java-backend && mvn clean package -DskipTests

# 4. 构建 Docker 镜像
docker build -t sjzm-backend:latest ./java-backend
docker build -t sjzm-frontend:latest ./frontend

# 5. 导入环境变量
cp .env.production .env

# 6. 启动服务
docker-compose -f docker-compose.yml -f docker-compose.prod.yml up -d

# 7. 检查服务状态
docker-compose ps
docker-compose logs -f backend
```

### 7.3 部署验证

```bash
# 1. 健康检查
curl http://localhost:8080/actuator/health
curl http://localhost:8080/actuator/prometheus

# 2. API 测试
curl -X POST http://localhost:8080/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"test123"}'

# 3. 数据库连接
curl http://localhost:8080/actuator/health/db

# 4. Redis 连接
curl http://localhost:8080/actuator/health/redis

# 5. 监控检查
curl http://localhost:9090/-/healthy
curl http://localhost:3000/api/health
```

## 八、回滚方案

### 8.1 快速回滚

```bash
# 1. 查看历史版本
docker images | grep sjzm-backend

# 2. 回滚到上一个版本
docker stop sjzm-backend
docker rm sjzm-backend
docker run -d --name sjzm-backend \
  --env-file .env \
  sjzm-backend:previous-version

# 3. 验证回滚
curl http://localhost:8080/actuator/health
```

### 8.2 数据库回滚

```bash
# 1. 恢复最近备份
gunzip < /data/backup/mysql/sijuelishi_YYYYMMDD_HHMMSS.sql.gz | \
  mysql -h localhost -u root -p sijuelishi

# 2. 验证数据
mysql -h localhost -u root -p sijuelishi -e \
  "SELECT COUNT(*) FROM product; SELECT COUNT(*) FROM user;"
```
