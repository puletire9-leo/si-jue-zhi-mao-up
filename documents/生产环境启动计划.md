## 三、实施步骤

### 3.1 第一步：清理旧容器
```bash
# 停止并删除所有旧容器
docker ps -a --filter "name=sjzm*" --filter "name=deploy-*" -q | xargs -r docker rm -f

# 清理停止的容器
docker container prune -f

# 可选：清理未使用的镜像
docker image prune -a -f
```

### 3.2 第二步：检查环境变量
需要验证 .env 文件中的关键配置：
- `MYSQL_PASSWORD` - 数据库密码
- `JWT_SECRET` - JWT 密钥
- `SECRET_KEY` - 加密密钥
- `COS_*` - 腾讯云配置（如果需要）

### 3.3 第三步：创建必要目录
```bash
# 在项目根目录创建
mkdir -p logs data
mkdir -p mysql/init mysql/conf
mkdir -p redis
mkdir -p rocketmq
```

### 3.4 第四步：启动基础服务（生产模式）
```bash
# 启动 MySQL + Redis + Qdrant
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d mysql redis qdrant

# 等待健康检查
docker compose -f docker-compose.yml -f docker-compose.prod.yml ps
```

### 3.5 第五步：初始化数据库
检查数据库初始化脚本是否在 `mysql/init/` 目录，如果没有需要创建。

### 3.6 第六步：启动后端服务
```bash
# 启动 Java 后端
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d java-backend

# 启动 Python 后端
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d backend

# 启动 Celery Worker
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d celery-worker
```

### 3.7 第七步：启动前端
```bash
# 启动前端
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d frontend
```

### 3.8 第八步：验证服务
```bash
# 检查所有服务状态
docker compose -f docker-compose.yml -f docker-compose.prod.yml ps

# 健康检查
curl http://localhost:8080/actuator/health
curl http://localhost:8000/health
curl http://localhost:80
```

## 五、潜在问题与解决方案

### 5.1 端口冲突
如果 8080/8000/80 端口被占用：
```bash
# 检查端口占用
netstat -ano | findstr :8080
netstat -ano | findstr :8000
netstat -ano | findstr :80
```

### 5.2 资源不足
生产配置要求：
- Java 后端：4 CPU + 4GB 内存
- Python 后端：2 CPU + 4GB 内存
- Celery：2 CPU + 2GB 内存

### 5.3 数据库初始化
需要检查 `/mysql/init/` 目录是否存在初始化 SQL 文件。
